# TABtransporter 작업 기준

## 1. 제품 목표

TABtransporter는 기타와 베이스 기타 악보를 입력받아 조옮김하고, 조옮김 후 실제 연주 가능한 TAB 운지를 다시 배치하는 무료 데스크톱 앱이다.

개념적으로 조옮김은 pitch를 이동하는 문제다.

지금 우리가 만드는 앱의 핵심은 이동된 pitch를 기타와 베이스에서 실제로 칠 수 있는 string/fret으로 다시 배치하는 것이다.

한 가지 직관:

음은 하나여도 손 위치는 여러 개다. 그래서 이 앱은 정답 하나를 찍는 도구가 아니라, 가능한 운지를 만들고 사람이 고르기 쉽게 해주는 도구로 만든다.

## 2. 절대 조건

- MVP라는 말을 기준으로 기능을 줄이지 않는다.
- 최종 목표는 모든 핵심 기능을 포함한 완성형 앱이다.
- 구현 순서는 나눌 수 있지만, 제품 범위에서 기능을 빼지 않는다.
- 기타와 베이스 기타를 모두 지원한다.
- 모든 핵심 기능은 무료 라이브러리와 무료 도구만 사용한다.
- 유료 API, 유료 SDK, 유료 변환 서버에 의존하지 않는다.
- Guitar Pro 앱은 유료/상용 소프트웨어이므로 제품 의존성, 필수 설치 도구, 권장 작업 흐름에서 제외한다.
- GP 계열 파일 자체는 무료 라이브러리로 가능한 범위에서 지원한다.
- 사용자가 실제로 넣을 파일은 PDF, 이미지, MusicXML, GP 계열 파일, MIDI, TAB 텍스트까지 포함한다고 본다.
- 사용자의 악보 파일은 기본적으로 로컬에서 처리한다.
- 자동 결과를 정답처럼 숨기지 않는다. 애매한 운지는 반드시 사용자가 확인할 수 있게 표시한다.
- 요청받지 않은 범위의 파일이나 기능은 임의로 바꾸지 않는다.
- 매번 작업은 구현, 검증, commit, git push까지 끝내는 것을 완료 기준으로 한다.
- git repository, remote, 인증, 네트워크 문제로 push가 불가능하면 완료로 말하지 말고 정확한 사유를 남긴다.

## 3. 앱 형태

- 앱 형태는 데스크톱 앱이다.
- Windows에서 실행 가능한 `.exe` 배포를 1순위로 둔다.
- 가능하면 macOS와 Linux도 나중에 같은 코드베이스로 빌드 가능해야 한다.
- 기본 동작은 오프라인 로컬 처리다.
- 네트워크가 필요한 기능은 핵심 기능으로 두지 않는다.
- 사용자는 별도 유료 프로그램 없이 앱 안에서 작업을 끝낼 수 있어야 한다.

## 4. 기술 선택

기본 구현 언어는 TypeScript로 간다.

앱 껍데기는 Electron을 1순위로 둔다.

이유:

- alphaTab이 TypeScript/JavaScript 환경에서 GP 계열 파일, MusicXML, TAB 렌더링, 재생 기능을 지원한다.
- Electron은 Windows `.exe` 배포, 로컬 파일 접근, 로컬 OMR 프로세스 실행에 적합하다.
- Tonal.js 같은 무료 음악 이론 라이브러리를 함께 쓰기 좋다.
- React/Vite 기반 UI를 그대로 데스크톱 앱에 넣을 수 있다.

기본 스택:

- App: TypeScript
- Desktop shell: Electron
- UI: React
- Build: Vite 계열
- Packaging: Windows `.exe` 생성이 가능한 무료 패키징 도구
- 악보/TAB 로딩, 렌더링, 재생: alphaTab 우선
- 음 이름, MIDI 번호, interval, 조옮김 계산: Tonal.js 또는 자체 순수 함수
- MusicXML 렌더링 보조: OpenSheetMusicDisplay 후보
- 직접 TAB/악보 렌더링 보조: VexFlow 후보
- PDF/이미지 악보 인식: 무료 로컬 OMR 엔진
- OMR 1순위 후보: Audiveris

기준:

- alphaTab으로 가능한 것은 alphaTab을 먼저 쓴다.
- alphaTab이 편집, 내보내기, 세밀한 운지 수정에서 부족하면 내부 악보 모델과 변환 로직으로 보완한다.
- 특정 기능에 더 강한 무료 라이브러리가 확인되면 언어 선택을 다시 검토한다.
- 무료 라이브러리라도 라이선스가 앱 배포에 영향을 주면 반드시 분리 구조를 검토한다.

## 5. 지원 입력 형식

입력은 폭넓게 지원한다.

필수 입력:

- PDF
- 이미지: PNG, JPG, JPEG, WebP, TIFF
- MusicXML: `.xml`, `.musicxml`, `.mxl`
- Guitar Pro 계열: `.gp3`, `.gp4`, `.gp5`, `.gpx`, `.gp`
- alphaTex
- MIDI: `.mid`, `.midi`
- 일반 텍스트 TAB

추가 호환 후보:

- Power Tab: `.ptb`
- TuxGuitar: `.tg`
- TablEdit: `.tef`
- CapXML: `.cap`

입력 처리 원칙:

- PDF와 이미지는 OMR로 구조화된 악보 데이터로 바꾼 뒤 처리한다.
- MusicXML과 GP 계열 파일은 가능한 한 직접 import한다.
- MIDI는 pitch와 rhythm은 가져오되 원래 운지가 없을 수 있으므로 기타/베이스 TAB 후보를 새로 생성한다.
- 일반 텍스트 TAB은 parser를 따로 둔다. 애매한 구문은 사용자가 수정하게 한다.
- import가 불완전하면 실패로 끝내지 말고 원본과 변환 결과를 비교할 수 있게 보여준다.

## 6. PDF/이미지 OMR

PDF/이미지 입력은 앱의 핵심 기능이다.

흐름:

1. 사용자가 PDF 또는 이미지 업로드
2. 앱이 무료 로컬 OMR 변환을 실행
3. MusicXML 또는 내부 악보 모델 생성
4. 원본과 변환 결과를 나란히 표시
5. 사용자가 인식 오류를 수정
6. 조옮김 실행
7. 기타/베이스 TAB 운지 재배치
8. 재생으로 확인
9. 최종 export

OMR 오류 수정은 가능한 범위를 모두 제공한다.

- 음표 위치 수정
- 음높이 수정
- 박자/마디 오류 수정
- 쉼표 누락 수정
- 임시표 누락 수정
- 코드명 오류 수정
- 가사 위치 오류 수정
- TAB 숫자 수정
- 줄/string 수정
- 프렛/fret 수정
- 튜닝 인식 오류 수정
- 마디 단위 재인식 또는 수동 입력

주의:

- OMR 결과는 완벽하지 않다.
- 앱은 인식 확신이 낮은 위치를 표시해야 한다.
- 원본 PDF/이미지와 변환 악보를 항상 비교할 수 있어야 한다.
- Audiveris는 AGPL 계열 라이선스이므로 직접 번들, 별도 프로세스, 사용자가 설치한 로컬 도구 연동 중 배포에 안전한 방식을 선택한다.
- 사용자가 보기에는 앱 안에서 PDF/이미지 변환이 진행되어야 한다.

## 7. 내부 악보 모델

앱은 외부 파일 형식에 직접 끌려가지 않는 내부 모델을 가진다.

필수 엔티티:

- Score
- Part
- Track
- Measure
- Voice
- Beat
- Note
- Rest
- ChordSymbol
- Lyric
- Technique
- TimeSignature
- KeySignature
- Tempo
- Tuning
- Instrument
- FingeringCandidate
- FingeringWarning

Note가 가져야 할 핵심 값:

- pitch 또는 MIDI number
- duration
- onset
- string
- fret
- tied/slur 여부
- articulation
- technique
- originalSource 위치
- recognitionConfidence

원칙:

- pitch와 TAB 위치를 분리해서 저장한다.
- 같은 음이 여러 string/fret 후보를 가질 수 있어야 한다.
- 원본 값, 변환 값, 사용자가 수정한 값을 구분한다.
- 사용자가 수정한 값은 자동 재계산으로 덮어쓰지 않는다.

## 8. 악기 모델

기타와 베이스를 별도 하드코딩하지 않는다. 둘 다 `fretted instrument`로 다룬다.

악기 설정은 다음 값을 가진다.

- instrumentType: guitar 또는 bass
- stringCount
- tuning
- fretCount
- capo
- preferredPosition
- allowedFretRange
- handStretchLimit
- preferredStringSet
- allowOpenStrings
- preserveOriginalStringBias

필수 프리셋:

- 6현 기타 표준 튜닝
- 7현 기타 표준 튜닝
- 12현 기타 기본 프리셋
- 4현 베이스 표준 튜닝
- 5현 베이스 표준 튜닝
- 6현 베이스 기본 프리셋
- 드롭 D 기타
- 반음 다운 기타
- 커스텀 튜닝

주의:

- 베이스는 기타보다 줄 수, 음역, 포지션 판단이 다르다.
- 베이스를 기타 TAB의 축소판처럼 처리하지 않는다.
- 베이스는 낮은 음역 보존, 포지션 이동, 줄 이동 비용을 기타와 다르게 점수화한다.

## 9. 조옮김 기능

조옮김은 항상 두 단계로 나눈다.

1. Pitch transpose
   - 모든 음의 실제 pitch를 목표 키나 interval만큼 이동한다.

2. Fingering remap
   - 바뀐 pitch를 현재 악기 설정에서 가능한 string/fret 후보로 변환한다.
   - 후보 중 실제 연주성이 좋은 것을 점수화해서 제안한다.

조옮김 입력 방식:

- 목표 키 선택
- 반음 단위 올림/내림
- interval 선택
- 현재 키와 목표 키 선택
- 카포 위치 기준 변환
- 튜닝 변경 기준 변환
- 선택 구간만 조옮김
- 트랙별 조옮김
- 전체 곡 조옮김

후보 생성 규칙:

- 각 줄의 개방현 pitch를 기준으로 fret을 계산한다.
- 계산된 fret이 악기 설정의 fretCount 안에 있어야 한다.
- capo가 있으면 capo 위치를 반영한다.
- 같은 시점의 여러 음이 같은 줄을 동시에 쓰면 불가능 후보로 본다.
- 악기 음역 밖의 pitch는 경고한다.

점수화 기준:

- 원래 줄과 가까운 후보를 우선한다.
- 이전 음과 포지션 이동이 적은 후보를 우선한다.
- 손가락 벌림이 지나치게 큰 후보는 낮게 본다.
- 개방현 느낌이 중요한 구간은 가능한 한 보존한다.
- 슬라이드, 해머링, 풀링, 벤딩 같은 주법은 가능한 한 유지한다.
- 빠른 구간에서 큰 포지션 점프가 생기면 경고한다.
- 같은 프레이즈 안에서는 포지션 일관성을 우선한다.
- 베이스는 낮은 줄의 역할과 루트 움직임을 고려한다.

중요:

- 자동 운지는 "가능한 운지 제안"이다.
- 최종 운지는 사용자가 듣고 직접 확인해야 한다.
- 앱은 확신이 낮은 마디를 표시해야 한다.
- 후보는 1개만 숨기지 말고 여러 후보를 비교할 수 있어야 한다.
- 사용자는 TAB 숫자, string, fret을 직접 수정할 수 있어야 한다.

## 10. 카포와 튜닝

카포는 조옮김 계산 옵션으로 제공한다.

지원:

- 카포 위치 선택
- 카포 적용 후 실제 sounding pitch 계산
- 카포 적용 상태에서 가능한 string/fret 재계산
- 원곡 손모양 유지 모드
- 원곡 손모양 유지와 실제 조옮김 TAB 비교
- 카포 사용이 더 나은 경우 추천

튜닝 지원:

- 표준 튜닝
- 드롭 튜닝
- 반음/온음 다운 튜닝
- 베이스 5현/6현 튜닝
- 사용자 커스텀 튜닝
- 튜닝 변경 후 기존 TAB 재계산

주의:

- 카포는 기타에서 주요 기능으로 노출한다.
- 베이스도 기술적으로 카포 값을 가질 수 있게 하되, 기본 UI에서는 필요할 때만 보이게 한다.

## 13. 출력과 저장

필수 출력:

- PDF
- PNG
- JPG
- MusicXML
- MIDI
- 앱 프로젝트 파일
- alphaTex 또는 내부 텍스트 포맷

가능한 범위에서 지원:

- GP 계열 export
- SVG
- WAV 또는 오디오 렌더링

저장 규칙:

- export 시 저장 위치를 사용자가 선택할 수 있어야 한다.
- 기본 저장 위치는 바탕화면이다.
- 앱 내부 임시 파일은 사용자에게 보이지 않는 앱 데이터 또는 임시 폴더에 둔다.
- 최종 산출물은 사용자가 고른 위치에 저장한다.
- 원본 파일은 절대 덮어쓰지 않는다.
- 자동 저장과 복구 파일을 제공한다.

## 14. UI와 디자인 기준

디자인은 음악 작업 도구처럼 보여야 한다.

피해야 할 것:

- AI 서비스 느낌의 과한 그라디언트
- 큰 랜딩 페이지
- 추상적인 장식 이미지
- 기능 설명을 길게 늘어놓는 화면
- 카드가 과하게 많은 마케팅형 레이아웃
- "AI가 알아서 해줌" 같은 문구

권장 방향:

- 첫 화면부터 파일 열기와 악보 작업 화면을 보여준다.
- 좌측은 파일, 페이지, 트랙, 악기 설정으로 둔다.
- 중앙은 원본 PDF/이미지와 변환 악보/TAB 비교 화면으로 둔다.
- 우측은 조옮김, 카포, 튜닝, 운지 옵션, 인식 오류 목록으로 둔다.
- 하단은 재생 컨트롤과 현재 위치 표시로 둔다.
- 색은 절제된 중립 톤을 기본으로 하고, 경고와 선택 상태만 명확한 색을 쓴다.
- 버튼은 아이콘과 짧은 텍스트를 쓴다.
- 악보, TAB, 재생 컨트롤, 조옮김 옵션이 한 화면에서 자연스럽게 연결되어야 한다.
- 화면은 장시간 작업해도 피로하지 않아야 한다.
- 텍스트는 짧고 실무 도구처럼 쓴다.

## 17. 테스트 기준

필수 테스트:

- PDF 샘플을 입력했을 때 OMR 변환 단계가 동작하는지
- 이미지 샘플을 입력했을 때 OMR 변환 단계가 동작하는지
- OMR 실패나 낮은 확신 결과를 사용자에게 숨기지 않는지
- MusicXML import가 동작하는지
- GP3/GP4/GP5/GPX/GP import가 가능한 범위에서 동작하는지
- MIDI import 후 TAB 후보 생성이 되는지
- 텍스트 TAB import가 동작하는지
- 기타 6현 표준 튜닝에서 조옮김이 맞는지
- 기타 7현에서 낮은 음역이 처리되는지
- 베이스 4현 표준 튜닝에서 조옮김이 맞는지
- 베이스 5현에서 낮은 음역이 유지되는지
- 같은 pitch가 여러 줄에서 나올 때 후보가 모두 생성되는지
- fretCount 밖의 음이 거부되는지
- 같은 박자에 같은 줄을 동시에 쓰는 후보가 거부되는지
- capo 적용 결과의 실제 sounding pitch가 맞는지
- 원곡 손모양 유지 모드와 실제 조옮김 모드가 구분되는지
- 재생 위치가 악보/TAB 표시와 동기화되는지
- PDF export가 열리는 파일로 생성되는지
- 이미지 export가 열리는 파일로 생성되는지
- MusicXML export가 다시 import 가능한지
- MIDI export가 재생 가능한지
- 저장 위치 기본값이 바탕화면인지
- 사용자가 저장 위치를 바꿀 수 있는지
- 원본 파일을 덮어쓰지 않는지
- Windows `.exe` 빌드가 생성되는지

테스트 원칙:

- 음정 계산은 숫자로 검증한다.
- UI는 실제 화면에서 겹침과 잘림을 확인한다.
- 렌더링 라이브러리가 만든 결과도 최소 샘플로 직접 확인한다.
- PDF/이미지 인식 결과는 원본과 비교할 수 있어야 한다.

## 18. 무료 도구와 라이브러리 기준

앱 내부 1순위 후보:

- alphaTab
  - 무료 MPL-2.0
  - TypeScript/JavaScript 중심
  - GP3-8, MusicXML, alphaTex 로딩
  - 표준보, 기타 TAB, 드럼 TAB 렌더링
  - 재생 기능 제공

- Tonal.js
  - 무료 MIT
  - TypeScript/JavaScript 음악 이론 라이브러리
  - note, interval, MIDI, transpose 계산에 적합

앱 내부 보조 후보:

- OpenSheetMusicDisplay
  - 무료 BSD-3-Clause
  - MusicXML 브라우저 렌더링에 강함
  - 편집기는 아니므로 렌더링 후보로 둔다.

- VexFlow
  - 무료 MIT
  - 악보와 기타 TAB을 직접 그릴 수 있는 저수준 렌더링 라이브러리
  - MusicXML 파서는 아니므로 직접 모델을 그릴 때만 후보로 둔다.

- PyGuitarPro
  - 무료 LGPL-3.0
  - Python에서 GP3/GP4/GP5 읽기, 쓰기, 조작 가능
  - GP 파일 호환성을 보강할 때 후보로 둔다.

- music21
  - 무료 BSD-3-Clause
  - Python 음악 분석/변환 라이브러리
  - MusicXML과 transpose 처리에 강함
  - TAB 운지 전용은 아니므로 보조 엔진으로만 둔다.

앱 외부 무료 검수 도구:

- Audiveris
  - 무료, 오픈소스, AGPL-3.0
  - PDF/이미지 악보를 MusicXML로 변환하는 OMR 도구
  - 인식률은 완벽하지 않으므로 변환 후 검수 필요

- MuseScore Studio
  - 무료, 오픈소스, GPL-3.0
  - MusicXML import/export, PDF export, TAB 편집 가능

- TuxGuitar
  - 무료, 오픈소스, LGPL
  - 멀티트랙 기타 TAB 편집/재생
  - GP, Power Tab, TablEdit, MIDI 파일 검수 후보

- Power Tab Editor
  - 무료, 오픈소스, GPL-3.0
  - PTB와 GP 계열 파일 검수 후보

- LilyPond
  - 무료, GPL
  - 고품질 악보 engraving 후보

제외 대상:

- Guitar Pro 앱
  - 유료/상용 소프트웨어
  - 제품 의존성으로 쓰지 않는다.

- Flat.io, Soundslice 같은 웹 서비스
  - 무료 제한 또는 유료 기능이 섞임
  - 핵심 기능 의존성으로 쓰지 않는다.

## 19. 라이선스 원칙

- 무료여도 라이선스는 반드시 확인한다.
- MIT, BSD, MPL 계열은 앱 내부 의존성으로 우선 검토한다.
- GPL, LGPL, AGPL 계열은 배포 구조를 따로 검토한다.
- AGPL 코드는 앱 본체에 직접 섞지 않는다.
- AGPL 도구가 필요하면 별도 로컬 프로세스, 사용자가 설치한 도구 연동, 또는 명확히 분리된 helper 구조를 검토한다.
- 유료 기능이 필요한 라이브러리는 핵심 경로에서 제외한다.
