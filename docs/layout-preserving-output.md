# 원본 레이아웃 보존 출력 기준

## 핵심 기준

최종 변환 결과는 새 카드형 악보로 다시 그리면 안 된다.

원본 PDF/이미지의 페이지 구성, 줄 간격, 마디 배치, 악보/TAB 위치를 최대한 그대로 유지하고, 실제로 바뀐 음표와 TAB 운지만 변환되어야 한다.

## 현재 구현 상태

현재 `TabPreview`와 `buildExportHtml`은 검수용 단순 렌더링이다.

이 화면은 내부 음정/TAB 계산을 확인하기 위한 임시 표시일 뿐, 최종 PDF/PNG 출력 기준이 아니다.

## 변환 성공 기준

PDF/이미지 입력에서 성공이라고 말하려면 다음 조건을 만족해야 한다.

- OMR이 정식 MusicXML 또는 동등한 구조 악보 데이터를 생성해야 한다.
- 조옮김 후 note pitch와 TAB string/fret이 실제로 바뀌어야 한다.
- 최종 PDF/PNG는 원본 페이지 디자인을 기준으로 다시 출력되어야 한다.
- fallback note-head 복구 데이터는 완성 변환으로 취급하지 않는다.
- 원본 레이아웃을 유지할 수 없으면 `converted`로 처리하지 않는다.

## 구현 방향

1. OMR 결과에서 음표, TAB 숫자, 마디, staff, page 좌표를 보존한다.
2. 내부 모델에 원본 page 좌표와 bounding box를 저장한다.
3. 조옮김과 운지 재배치 후 원본 좌표 위에 변환된 기호를 배치한다.
4. 기존 기호를 가리거나 대체하는 레이어를 만든다.
5. PDF/PNG export는 이 레이아웃 보존 레이어를 기준으로 생성한다.

한 가지 직관:

새 악보를 예쁘게 다시 그리는 것이 아니라, 원본 악보 종이 위의 음표와 TAB 숫자만 바꿔 끼우는 방식이어야 한다.
